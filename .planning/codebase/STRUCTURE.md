## Directory Purposes

**`.planning/codebase/`:**
- Purpose: GSD framework outputs (ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, etc.)
- Contains: Markdown analysis documents
- Key files: (generated by gsd:map-codebase)

**`.github/workflows/`:**
- Purpose: CI/CD pipeline automation
- Contains: GitHub Actions YAML definitions
- Key files: `pipeline.yml` (versioner → lint → test → build → integration → release)

**`assets/`:**
- Purpose: Project branding and documentation images
- Contains: Icons, screenshots, logos

**`litellm/`:**
- Purpose: LiteLLM service configuration
- Key file: `config.yaml` — provider routing (maps claude-* to Anthropic, gpt-* to OpenAI, etc.)

**`nginx/`:**
- Purpose: Reverse proxy for Docker services
- Key file: `nginx.conf` — routes localhost ports to container DNS names

**`pipelines/`:**
- Purpose: Two-stage inference pipeline implementations
- Contains: OpenWebUI Pipe objects (async streaming, Valves config)
- Structure: Each pipeline is a `Pipeline` class with `pipe()` async method
- Key files:
  - `code_review_pipe.py` — Qwen coder (stage 1) → Claude review (stage 2)
  - `reason_review_pipe.py` — DeepSeek reasoning (stage 1) → Claude structure (stage 2)
  - `chat_assist_pipe.py` — Llama chat (stage 1) → Claude polish (stage 2)

**`searxng/`:**
- Purpose: Web search engine aggregation service
- Key file: `settings.yml` — search engine list and configuration

**`rthooks/`:**
- Purpose: PyInstaller runtime customization
- Key file: `pyi_rth_rich_unicode.py` — ensure Unicode rendering in frozen exe

**`scripts/`:**
- Purpose: Development utilities (not distributed with exe)
- Key file: `take_screenshots.py` — demo UI screenshots

**`tests/unit/`:**
- Purpose: Fast, isolated unit tests (no Docker dependency)
- Contains: Config, routing, tool execution, token budget, slash commands, streaming, UI, startup
- Run: `python -m pytest tests/unit/ -q --cov=clod`

**`tests/integration/`:**
- Purpose: Integration tests with real services (Docker-backed)
- Contains: Subprocess execution, exe testing
- Run: `python -m pytest tests/integration/ -q` (non-PR only in CI)

**`tools/`:**
- Purpose: External tool integrations (Claude.com MCP client)
- Status: Present but not used by CLI (may be deprecated)

## Naming Conventions

**Files:**

- `clod.py` — Main CLI (PascalCase for class, snake_case for functions)
- `mcp_server.py` — Supporting module (snake_case)
- `*_pipe.py` — Pipelines (snake_case with _pipe suffix)
- `test_*.py` — Test files (snake_case with test_ prefix)
- `*_extended.py` — Extended test coverage (snake_case with _extended suffix)

**Directories:**

- lowercase, plural or descriptive: `tests/`, `pipelines/`, `scripts/`, `assets/`, `tools/`
- service config: `litellm/`, `nginx/`, `searxng/` (lowercase service names)

**Functions:**

- snake_case: `load_config()`, `stream_ollama()`, `handle_slash()`
- Private (leading underscore): `_parse_dotenv()`, `_check_service_health()`, `_ensure_local_configs()`
- Tool handlers: `tool_<name>()` (e.g., `tool_bash_exec()`, `tool_read_file()`)

**Classes:**

- PascalCase: `TokenBudget`, `Pipeline` (in pipes)
- Inner class: `Valves` (pipeline config)

**Constants:**

- ALL_CAPS: `CLOUD_MODEL_PREFIXES`, `TOOL_DEFINITIONS`, `VRAM_TIERS`, `TOKEN_WARN`, `TOKEN_LIMIT`, `MCP_SERVER_PORT`
- All-caps dict names: `PIPELINE_CONFIGS`, `PROJECT_SIGNALS`, `SKIP_DIRS`, `CONTEXT_FILE_PATTERNS`

**Variables & Parameters:**

- snake_case: `session_state`, `tool_calls`, `final_content`, `base_url`, `max_tokens`
- Abbreviations: `cfg` (config), `gen` (generator), `fn` (function), `msg` (message), `resp` (response)

## Where to Add New Code

**New Feature (Model or Command):**

- Primary code: `clod.py` (likely a new function + integration in `infer()` or `handle_slash()`)
- Tests: `tests/unit/test_*.py` (new file or extend existing)
- Documentation: Update CLAUDE.md, README.md

**New Slash Command:**

- Handler: Add case in `handle_slash()` function (line 2079+), returns boolean
- Mutation: Update `session_state` dict or call helper functions
- Tests: `tests/unit/test_slash.py` or `test_slash_extended.py`
- Help: Update help text in `print_help()` (line 1898+)

**New Tool (bash, file, web search):**

- Definition: Add entry to `TOOL_DEFINITIONS` list (line 160+)
- Handler: Create `tool_<name>()` function (line 290+), returns string result
- Dispatcher: Update `execute_tool()` switch (line 384+)
- Tests: `tests/unit/test_tools.py` or `test_tools_extended.py`

**New Pipeline:**

- Implementation: Create `pipelines/<name>_pipe.py`
- Class: Inherit structure from `code_review_pipe.py`, define `Valves`, implement `pipe()` async method
- Config: Update `PIPELINE_CONFIGS` dict in `clod.py` (line 62+)
- Docker: Add service to `docker-compose.yml` (if needed, already has generic Pipelines service)

**New Configuration Key:**

- Default: Add to `load_config()` defaults (line 251+)
- Persistence: Load from `config.json` or `.env` (no code change needed)
- Usage: Pass via `cfg` dict to functions

**New Service Health Check:**

- Check function: Update `_check_service_health()` (line 825+), add HTTP ping
- Feature flag: Update `_compute_features()` (line 844+), add to returned dict
- Display: Update `print_startup_banner()` (line 1360+) health section

**Utilities & Helpers:**

- Shared helpers: Add to `clod.py` if general-purpose, or create `lib_*.py` if complex
- Test helpers: Add to `tests/conftest.py` (shared fixtures) or test file itself
