## Cross-Phase Integration Issues

### Priority 1 — Runtime Failure Risk

**Missing Python dependencies in requirements.txt**
- `numpy`, `onnxruntime`, `tokenizers` are imported by `intent.py` but absent from `requirements.txt`
- On clean install: `import intent` fails → `HAS_INTENT = False` → all intent classification silently disabled
- Affects: INTENT-01, INTENT-02, ROUTE-01 (downstream of classification availability)

### Priority 2 — Silent UX Failure

**`image_edit` intent not dispatched**
- `classify_intent` returns `("image_edit", 0.95)` for "edit this photo" inputs
- REPL interception at line 3590 only checks `("image_gen", "video_gen")` — `image_edit` falls through silently to `infer()`
- No error message, no "not supported" notice — user gets chat model response to an image editing request
- Affects: INTENT-01 (partial), ROUTE-01 (partial)

### Priority 3 — Quality Gate Gaps

- `intent.py` excluded from `black`, `pylint`, and coverage measurement in CI
- REPL loop dispatch path (`classify_intent` → interception → `_handle_generation_intent`) not directly tested (TTY limitation — acceptable)
- `query_comfyui_running()` function name misleading (checks AUTOMATIC1111, not ComfyUI)
