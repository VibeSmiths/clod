  <files>tests/unit/test_services_slash.py</files>
  <action>
Create `tests/unit/test_services_slash.py` with tests for all `/services` sub-commands via `handle_slash()`.

Use established patterns: `@responses.activate` for HTTP mocking, `FakeConsole` from conftest.py, `mock_session_state` fixture. Create a local `service_cfg` fixture that extends `mock_cfg` with `compose_file` and `dotenv_file` keys pointing to tmp_path files (per Pitfall 2 from RESEARCH.md).

**Tests to write:**

Status sub-command:
- `test_services_status_all_healthy`: mock all 5 health endpoints as 200, verify `handle_slash("/services", ...)` returns True and updates `session_state["health"]`
- `test_services_status_some_down`: mock ollama as 200, others as ConnectionError, verify health dict reflects mixed state

Start sub-command:
- `test_services_start_all_running`: all health checks pass, verify "All core services are running" path (no docker compose invoked)
- `test_services_start_missing_services`: some health checks fail, mock `_offer_docker_startup` to return True, verify `session_state["features"]` and `session_state["health"]` are updated
- `test_services_start_docker_declined`: mock `_offer_docker_startup` to return False, verify session_state unchanged

Stop sub-command:
- `test_services_stop_confirmed`: use `_InputConsole` pattern with input_val="y", mock `subprocess.run` for docker compose down success, verify `session_state["offline"]` set True and health all False
- `test_services_stop_cancelled`: input_val="n", verify no subprocess calls made
- `test_services_stop_compose_failure`: input_val="y", mock subprocess returncode=1 with stderr, verify error message

Reset sub-command:
- `test_services_reset_single_named`: `/services reset ollama`, mock subprocess for stop/rm/up, mock `_get_service_volumes` to return paths, verify `_reset_service` called for ollama only
- `test_services_reset_all`: `/services reset all`, use `_InputConsole` for delete mode prompt, verify all services reset in dependency order
- `test_services_reset_unknown_service`: `/services reset nonexistent`, verify "Unknown service" warning
- `test_services_reset_no_compose_file`: compose_file path doesn't exist, verify "not found" error

Mock `clod.subprocess.run` (NOT `subprocess.run` -- per Pitfall 4). Use `_InputConsole` pattern from test_startup.py for interactive prompts. Register all 5 health check URLs when testing start/status (per Pitfall 3).

For reset tests, monkeypatch `_reset_service` and `_get_service_volumes` instead of mocking subprocess directly -- test the routing logic, not the subprocess calls (those are tested in Task 2).
