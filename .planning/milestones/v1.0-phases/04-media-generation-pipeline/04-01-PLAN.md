Add to clod.py (near the GPU service functions, around line 815):

1. **Constants** (near top of file, after INTENT_MODEL_MAP):
   - `SD_PROMPT_SYSTEM` -- system prompt for llama3.1:8b to craft SD-optimized prompts. Rules: output ONLY the prompt as comma-separated tags, include quality boosters (masterpiece, best quality, highly detailed), include style/lighting terms. If user mentions exclusions ("but no people"), output on second line prefixed with "EXCLUDE:". Per user decision: single-shot, no refinement.
   - `SD15_NEGATIVES` -- verbose negative prompt string for SD1.5 (low quality, worst quality, ugly, blurry, extra limbs, extra fingers, mutated hands, poorly drawn hands/face, deformed, disfigured, watermark, text, signature, out of frame)
   - `SDXL_NEGATIVES` -- short negative prompt string for SDXL (low quality, worst quality, watermark, text, signature)
   - `SD_DEFAULT_PARAMS` -- dict with steps=25, cfg_scale=7, width=512, height=512, sampler_name="DPM++ 2M"
   - `SDXL_DEFAULT_PARAMS` -- dict with steps=25, cfg_scale=7, width=1024, height=1024, sampler_name="DPM++ 2M"

2. **`_parse_crafted_prompt(response_text: str) -> tuple[str, str]`**: Parse LLM response into (positive_prompt, user_exclusions). Split on "EXCLUDE:" line. Strip whitespace.

3. **`_craft_sd_prompt(user_request: str, cfg: dict) -> tuple[str, str]`**: Call Ollama /api/chat with model="llama3.1:8b", system=SD_PROMPT_SYSTEM, user=user_request, stream=False. Parse response via _parse_crafted_prompt. On any exception, return (user_request, "") as graceful fallback.

4. **`_detect_sd_model_type() -> str`**: GET SD_WEBUI_URL/sdapi/v1/options, extract sd_model_checkpoint, case-insensitive match for "sdxl", "xl", "pony" -> return "sdxl", else "sd15". Fallback "sd15" on any error.

5. **`_get_negative_prompts(model_type: str, user_exclusions: str = "") -> str`**: Return SDXL_NEGATIVES if model_type=="sdxl" else SD15_NEGATIVES, append user_exclusions if non-empty.

6. **`_save_generation_output(data: bytes, ext: str, output_dir: str) -> str`**: Create output_dir if needed. Generate filename: `clod_{YYYYMMDD_HHMMSS}_{md5(data[:256])[:4]}.{ext}`. Write bytes. Return filepath.

7. **`_generate_image(prompt: str, negative: str, params: dict, cfg: dict, console_obj) -> str | None`**: POST to SD_WEBUI_URL/sdapi/v1/txt2img in a background thread (threading.Thread). Main thread polls SD_WEBUI_URL/sdapi/v1/progress?skip_current_image=true every 1.5s with Rich Progress bar (TextColumn, BarColumn, percentage, TimeRemainingColumn). After thread joins, decode base64 image, save via _save_generation_output. Return filepath or None on error. Use save_images=False in payload (we save ourselves). Timeout 300s for the POST.
