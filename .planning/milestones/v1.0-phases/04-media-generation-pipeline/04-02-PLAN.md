1. **`VIDEO_PROMPT_SYSTEM`** constant (near SD_PROMPT_SYSTEM): System prompt for video generation. Rules: output ONLY the optimized prompt, describe motion/temporal progression, include camera movement if appropriate, specify style/mood, keep concise (1-3 sentences).

2. **`_craft_video_prompt(user_request: str, cfg: dict) -> str`**: Same pattern as _craft_sd_prompt but using VIDEO_PROMPT_SYSTEM. Returns single string (no exclusions for video). Fallback to user_request on error.

3. **`_build_video_workflow(prompt: str) -> dict`**: Construct a basic ComfyUI workflow JSON for text-to-video. Use a simple parameterized workflow template with the prompt injected into the positive prompt node. Include a comment noting the workflow should be customized per installed video model. Return the workflow dict.

4. **`_download_comfyui_output(item: dict, cfg: dict, console_obj) -> str | None`**: Download file from ComfyUI /view endpoint. Item has "filename", "subfolder", "type" keys. GET COMFYUI_URL/view?filename=...&subfolder=...&type=... Save to COMFYUI_OUTPUT_DIR with clod_{timestamp}_{hash}.{ext} naming. Return filepath or None.

5. **`_generate_video(prompt: str, cfg: dict, console_obj) -> str | None`**: POST workflow to COMFYUI_URL/prompt. Poll COMFYUI_URL/history/{prompt_id} every 3s with console_obj.status spinner (not progress bar -- ComfyUI doesn't expose step-level progress via HTTP). 10-minute timeout. When complete, find output node with "gifs" or "images" key, download via _download_comfyui_output. Return filepath or None.

6. **`_ensure_generation_service(intent: str, cfg: dict, console_obj, session_state: dict) -> bool`**:

   - For "video_gen": check query_video_running(). If running, return True. If not:
     - If query_comfyui_running() is True (image profile running): print warning "Switching from image to video mode. This will stop AUTOMATIC1111 and start ComfyUI." Ask confirmation via console_obj.input(). If declined, return False. If confirmed, call sd_switch_mode("video", cfg). Then call _prepare_for_gpu_service("comfyui", cfg, console_obj, session_state). (Covers DOCK-01, DOCK-02, DOCK-03, VID-03)
     - If nothing running: just call _prepare_for_gpu_service("comfyui", cfg, console_obj, session_state)
   - GPU release verification happens inside _prepare_for_gpu_service (already polls nvidia-smi). DOCK-03 is satisfied.
