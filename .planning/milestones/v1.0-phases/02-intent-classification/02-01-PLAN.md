```python
def _get_clod_root() -> pathlib.Path:
    """Return the directory that contains clod's runtime files.
    Frozen (PyInstaller exe): directory containing the exe.
    Script mode: directory containing clod.py.
    """
    if getattr(sys, "frozen", False):
        return pathlib.Path(sys.executable).parent
    return pathlib.Path(__file__).parent
```

  <action>
Create `intent.py` in the project root (alongside clod.py) following the architecture from 02-RESEARCH.md:

1. **Constants and types:**
   - `INTENTS` tuple: ("chat", "code", "reason", "vision", "image_gen", "image_edit", "video_gen")
   - Use section comments matching codebase style: `# -- Section Name ----`

2. **Layer 1 - _classify_keywords(text):**
   - Keyword/regex rules as list of (intent, compiled_pattern, confidence) tuples
   - Patterns from RESEARCH.md Pattern 1, with word boundaries and proximity constraints
   - Order: more specific intents first (image_gen/image_edit/video_gen before code/reason, since compound requests could match multiple)
   - Return Optional[Tuple[str, float]] -- None if no keyword match
   - Vision intent: keyword-only detection (e.g., "describe this image", "what's in this picture", "OCR this screenshot"). Do NOT attempt to detect image attachments -- the codebase has no image attachment mechanism.

3. **Layer 2 - IntentEmbedder class:**
   - `__init__(model_path, tokenizer_path)` -- loads ONNX session with CPUExecutionProvider, loads tokenizer from file
   - **CRITICAL:** Check `session.get_inputs()` to get input names. Only pass `token_type_ids` if it appears in the input list. Some ONNX exports omit this input.
   - `embed(text) -> np.ndarray` -- tokenize, run ONNX inference, mean pool with attention mask, L2 normalize, return (384,) vector
   - Enable truncation (max_length=256) and padding (pad_id=0, pad_token="[PAD]") on tokenizer
   - Follow RESEARCH.md Pattern 2 exactly

4. **Layer 2 - _classify_embedding(text):**
   - Uses module-level lazy-loaded embedder and route embeddings
   - Loads route_embeddings.npz from `_get_clod_root() / "models" / "intent" / "route_embeddings.npz"`
   - Cosine similarity via dot product against centroid matrix (7 dot products, sub-1ms)
   - Return (intent, confidence) for best match

5. **Lazy loading:**
   - Module-level `_embedder: Optional[IntentEmbedder] = None` and `_route_centroids` / `_route_intent_names`
   - `_ensure_embedder()` function that initializes on first call
   - Import `_get_clod_root` from clod module inside the function to avoid circular imports

6. **Public API - classify_intent(text, threshold=0.8):**
   - Try keyword rules first (fast path)
   - If no keyword match, try embedding similarity
   - Return (intent, confidence) tuple always
   - If all below threshold, return the best embedding match with its actual confidence (caller decides)

7. **cosine_similarity(query, matrix):**
   - `matrix @ query` for L2-normalized vectors
   - Returns np.ndarray of shape (n_routes,)

8. **Create `models/intent/build_routes.py`:**
   - Script with ROUTES dict (seed utterances per intent from RESEARCH.md Pattern 4)
   - `build_centroids(embedder, routes, output_path)` function: embeds all utterances, computes per-intent centroid mean, **re-normalizes centroids after averaging** (Pitfall 7), saves as .npz
   - Include `if __name__ == "__main__"` block

9. **Tests in `tests/unit/test_intent.py`:**
   - Mock the ONNX session and tokenizer for embedding tests (do NOT require real model files)
   - Create fixtures: `mock_onnx_session`, `mock_tokenizer`, `mock_route_embeddings`
   - Test keyword classification directly with known inputs
   - Test embedding path with mocked embedder returning controlled 384-dim vectors
   - Test classify_intent integration with both layers
   - Test cosine_similarity math
   - Test latency of keyword-only path (< 100ms, should be < 1ms)
   - Use monkeypatch to mock `_get_clod_root` for path resolution tests

IMPORTANT: Use `onnxruntime` and `tokenizers` libraries only -- NOT sentence-transformers or torch. PyTorch bloat is unacceptable (would balloon exe from ~50MB to ~1.8GB).
