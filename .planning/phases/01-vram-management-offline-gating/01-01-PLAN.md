    ```
    # ── VRAM Management ──────────────────────────────────────────────────────────
    ```

    Implement these functions following the patterns from RESEARCH.md:

    1. `_get_loaded_models(cfg: dict) -> list[dict]` -- GET /api/ps, return models list, [] on error
    2. `_unload_model(model_name: str, cfg: dict) -> bool` -- POST /api/generate with keep_alive:0 and empty prompt, return success bool
    3. `_verify_vram_free(min_free_mb: int = 4000) -> bool` -- call query_gpu_vram(), return True if free >= threshold OR if gpu is None
    4. `_vram_transition_panel(phase: str, console_obj) -> None` -- print "VRAM: X.X/Y.Y GB (phase)" using query_gpu_vram(). No-op if None.
    5. `_ensure_model_ready(target: str, cfg: dict, console_obj, session_state: dict) -> bool`:
       - Check /api/ps: if target already loaded, return True
       - Unload all loaded models via _unload_model
       - Show VRAM panel "Unloading..."
       - Poll _verify_vram_free for up to 10s (5 iterations, 2s sleep)
       - Show VRAM panel "Loading..."
       - Check if model needs pulling (ensure_ollama_model) or just warmup (warmup_ollama_model)
       - Show VRAM panel "Ready"
       - Update session_state["model"] = target
       - Return True
    6. `_prepare_for_gpu_service(cfg: dict, console_obj) -> bool`:
       - Get loaded models, if none return True immediately
       - Print "Freeing GPU for generation service..."
       - Unload all models
       - Poll nvidia-smi via _verify_vram_free for up to 15s (8 iterations, 2s sleep), checking that free_mb > (total_mb - 2000)
       - Show VRAM panel "GPU freed" on success
       - Warn "VRAM may not be fully freed. Proceeding anyway." on timeout
       - Return True

    Error recovery for _unload_model failures (Claude's discretion): retry once after 3s wait. If still fails, warn user and return False. Do NOT attempt Ollama restart in this phase.

    When nvidia-smi unavailable: skip VRAM verification, log "[dim]GPU monitoring unavailable -- skipping VRAM verification[/dim]", proceed optimistically.

    Run tests (expect GREEN).
